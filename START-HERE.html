1

Make the database (Supabase)

~5 min

This is the box that holds all your jobs. It's free.

  1. Go to supabase.com and sign up (use your Google login — fastest).
  2. Click "New Project." Give it a name like orbital, pick any password, hit Create. Wait ~2 min while it builds.Grab a coffee — it's setting up your tank.
  3. Open the SQL Editor (left sidebar icon that looks like a database). Click "New query."
  4. Copy the code below, paste it in, and press "Run." This builds the shelves inside the box. You should see "Success."
paste this into Supabase → SQL Editor → Run
-- Orbital: build the jobs table. Safe to run more than once.
create table if not exists public.jobs (
  idx             bigint generated always as identity,
  id              text primary key,
  company         text not null,
  title           text not null,
  location_hub    text,
  location        text,
  salary_range    text,
  salary_min      int,
  salary_max      int,
  intern_rate     int,
  intern_weeks    int,
  is_intern       boolean default false,
  type            text,
  match           int,
  planets         int,
  demand          int,
  posting_age     int,
  tech            text,
  clearance       text,
  relocation_flag boolean default false,
  relocation_note text,
  logo_bg         text,
  url             text,
  timestamp       timestamptz,
  description     text,
  requirements    text,
  source          text,
  status          text default 'active',
  saved           boolean default false,
  archived        boolean default false,
  created_at      timestamptz default now()
);

-- let the website READ jobs (but never let strangers change them)
alter table public.jobs enable row level security;
drop policy if exists "anon read jobs" on public.jobs;
create policy "anon read jobs" on public.jobs
  for select to anon using (true);

-- grant write access so the service_role key can save jobs (fixes "permission denied" /
--    "violates row-level security policy" if you ever see either of those)
grant usage on schema public to anon, authenticated, service_role;
grant select on public.jobs to anon, authenticated;
grant all privileges on public.jobs to service_role;
grant insert, update, delete on public.jobs to service_role;
-- single-user tool: disable row-level security so the service_role key can always write
alter table public.jobs disable row level security;
drop policy if exists "service role writes" on public.jobs;
create policy "service role writes" on public.jobs
  for all to service_role using (true) with check (true);

create index if not exists jobs_hub_time_idx on public.jobs (location_hub, timestamp desc);
  1. Get your 2 keys. Go to Settings → API. You'll need three things — keep this tab open for Step 3:
    Project URL
    looks like https://abcd.supabase.co
    anon public key
    long code starting "eyJ…"
    service_role key
    another long "eyJ…" (the secret one)

2

Turn on the daily auto-feeder (GitHub)

~5 min

This robot pulls fresh space jobs into your database every day — for free, while you sleep.

  1. Go to github.com and make a free account (or log in).
  2. Click the "+" top-right → "New repository." Name it orbital. Choose Private. Click "Create repository."A "repository" is just a folder that lives online.
  3. Upload the Orbital files. On the new repo page click "uploading an existing file," then open the orbital-crm folder you downloaded, select everything INSIDE it, and drag those items in. Click "Commit changes."
    Upload the CONTENTS, not the folder itself
    You want these files at the top level of your repo (so index.html is right there — not buried inside an orbital-crm/ subfolder, or the robot can't find .github):
    index.html
    setup.html
    START-HERE.html
    README.txt
    manifest.json
    orbital-logo.png
    scraper.py
    requirements.txt
    📁 assets/  (headshot.png, resume.pdf)
    📁 .github/  (hidden — workflows/scrape.yml + deploy.yml)
    Tip: open the orbital-crm folder, press Ctrl+A (Cmd+A on Mac) to select all, and drag the highlighted files onto the GitHub page.
    ⚠ Didn't see the .github folder? It's hidden — create it by hand (2 min)
    The .github folder starts with a dot, so your computer hides it — and without it the automation can't run. Easiest fix: make the files directly on GitHub, which also creates the folders for you.
    1. In your repo: Add file ▾ → Create new file.
    2. In the filename box type exactly: .github/workflows/scrape.yml (the slashes auto-create the folders)
    3. Paste the Scraper workflow below → Commit changes.
    4. Repeat: Add file → Create new file → .github/workflows/deploy.yml → paste the Website-publish workflow → Commit.
    .github/workflows/scrape.yml
    name: Orbital daily scrape
on:
  schedule:
    - cron: "0 12 * * *"
  workflow_dispatch: {}
jobs:
  scrape:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"
      - name: Install dependencies
        run: pip install -r requirements.txt
      - name: Run scraper
        env:
          SUPABASE_URL: ${{ secrets.SUPABASE_URL }}
          SUPABASE_SERVICE_ROLE_KEY: ${{ secrets.SUPABASE_SERVICE_ROLE_KEY }}
        run: python scraper.py
      - name: Upload failure log (if any)
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: failed-scrapes-log
          path: failed_scrapes_log.csv
          if-no-files-found: ignore
    .github/workflows/deploy.yml
    name: Publish website to IONOS
on:
  push:
    branches: [ main, master ]
    paths:
      - 'index.html'
      - 'setup.html'
      - 'START-HERE.html'
      - 'manifest.json'
      - 'orbital-logo.png'
      - 'assets/**'
  workflow_dispatch: {}
jobs:
  publish:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Upload website to IONOS via FTP
        uses: SamKirkland/FTP-Deploy-Action@v4.3.5
        with:
          server: ${{ secrets.IONOS_FTP_SERVER }}
          username: ${{ secrets.IONOS_FTP_USERNAME }}
          password: ${{ secrets.IONOS_FTP_PASSWORD }}
          protocol: ftps
          exclude: |
            **/.git*
            **/.github/**
            scraper.py
            requirements.txt
            README.txt
          server-dir: ./
  4. Add your secret keys so the robot can write to your database. Go to Settings → Secrets and variables → Actions → New repository secret and add these two:
    SUPABASE_URL
    paste your Project URL from Step 1
    SUPABASE_SERVICE_ROLE_KEY
    paste your service_role key from Step 1
    Type the names EXACTLY as shown (all caps, with underscores).
  5. Run it once now. Go to the Actions tab → click "Orbital daily scrape" → "Run workflow." Wait ~2 minutes. Green check = real jobs are now in your database. 🎉
🤖
After this, it runs automatically every day. You never have to touch it again. To add fresh jobs anytime, just hit "Run workflow" again.

3

Connect IONOS once — then it auto-publishes

~5 min

Instead of ever uploading to IONOS by hand, you give GitHub your IONOS login once. After that, GitHub publishes the website to jobs.lisaney.com automatically — every time anything changes.

  1. Get your IONOS FTP login. In IONOS → your hosting → SFTP/FTP access. Note the server/host, username, and password (create a password if it doesn't show one).
  2. Add them to GitHub (same place as Step 2's secrets): your repo → Settings → Secrets and variables → Actions → New repository secret. Add these three, names EXACTLY:
    IONOS_FTP_SERVER
    your IONOS host (e.g. access-xxxx.webspace-data.io)
    IONOS_FTP_USERNAME
    your IONOS FTP username
    IONOS_FTP_PASSWORD
    your IONOS FTP password
  3. Publish it. Repo → Actions tab → “Publish website to IONOS” → Run workflow. In ~1 minute your site is live at jobs.lisaney.com. (From now on it re-publishes itself automatically whenever the repo changes.)
  4. Open jobs.lisaney.com/setup.html, paste your Supabase Project URL + anon key from Step 1, run the first fetch, and finish the GitHub gate. Log in with PIN 1725 to edit.
✅
Final check before you call it launched — go to your repo → Settings → Secrets and variables → Actions and confirm all FIVE secrets are listed (names must match exactly, all caps):
SUPABASE_URL
SUPABASE_SERVICE_ROLE_KEY
IONOS_FTP_SERVER
IONOS_FTP_USERNAME
IONOS_FTP_PASSWORD
GitHub only shows secret NAMES, never the values — that's expected, it means they're saved. If any are missing, add it (New repository secret) and re-run that workflow once.
🔌
Prefer to upload by hand instead? You can still skip the FTP secrets and just drag these files into your IONOS File Manager: index.html · setup.html · START-HERE.html · manifest.json · orbital-logo.png · assets/. But the auto-publish above means you never have to do that again.

If something feels stuck

The website opens but shows "No Data"
That just means the database is empty so far. Do the GitHub "Run workflow" (Step 2 #5) or hit "Fetch real jobs" on setup.html. Wait 2 minutes, refresh. Jobs will appear.
The GitHub run failed (red X)
Almost always a typo in the secret names. They must be SUPABASE_URL and SUPABASE_SERVICE_ROLE_KEY exactly — all caps, underscores, no spaces. Fix and run again. The run also leaves a "failed-scrapes-log" file you can download if you're curious which boards skipped.
"Fetch real jobs" says some companies were blocked
Totally normal. A few company boards block browsers for security. Those ones come in through the daily GitHub robot instead (Step 2). Between the two, you get everything reachable.
Supabase said "relation jobs already exists"
That's not an error you need to worry about — it just means the jobs table was already created. You're good; move to Step 2. The code is safe to paste and run again anytime without breaking anything.
I forgot my login
The owner PIN is 1725. Anyone without it just sees a read-only view — they can look but not change anything.
Do I have to do this every time?
No. This is a one-time setup. After that, the daily robot keeps it fresh forever and you just open jobs.lisaney.com.
